# Log Prompt - 102022400230_Tiara-Vehicle-Service & Integrasi Web

Dokumen ini berisi narasi log *prompt* (instruksi) yang digunakan secara bertahap untuk menyelesaikan pengembangan *backend service* kendaraan (`102022400230_Tiara-Vehicle-Service`) serta proses integrasinya dengan *frontend* Web Dispatching Armada secara end-to-end.

### 1. Inisialisasi Proyek & Skema Database
"Tolong buatkan struktur awal proyek Laravel untuk Tiara-Vehicle-Service. Setelah itu, bantu saya mengonfigurasi koneksi database di file `.env`. Tolong sekalian buatkan file *migration*, *seeder*, dan *model* untuk entitas `Vehicle` yang mencakup kolom: id, vehicle_code, plate_number, brand, model, status, dan capacity."

### 2. Pembuatan API CRUD Kendaraan
"Selanjutnya, tolong buatkan *Controller* API untuk `Vehicle` yang dapat meng-*handle* fungsionalitas GET (untuk memuat semua data), POST (untuk menambah kendaraan baru), PATCH (untuk meng-update data dan mengubah status kendaraan), serta DELETE untuk menghapus data armada. Pastikan rute *endpoint*-nya diarahkan ke `/api/v1/vehicles`."

### 3. Konfigurasi API Gateway
"Sekarang bantu saya mengatur API Gateway. Saya ingin agar semua *request* masuk dari aplikasi frontend yang mengarah ke rute `/api/v1/vehicles` diteruskan (*forwarded*) secara tepat ke layanan Tiara-Vehicle-Service yang berjalan di belakangnya, agar tidak ada *error* rute atau masalah *CORS*."

### 4. Integrasi Web Frontend & Data Dinamis
"Tolong hubungkan web frontend dengan API backend kendaraan yang sudah jadi. Pada file `app.js`, tolong ubah fungsi tombol 'Tambah Kendaraan' yang awalnya hanya men-*generate* data demo statis menjadi form *pop-up modal* manual. Pastikan form ini mengirim *request* POST yang sesungguhnya ke API, sehingga data baru benar-benar tersimpan ke *database*."

### 5. Perombakan Navigasi UX (Multi-tab)
"Ubah kerangka aplikasi *Single-Page* ini menjadi navigasi berbasis *multi-tab* di *sidebar* (menjadi halaman Overview, Buat Jadwal, Kendaraan, Jadwal Driver, dan Operasional). Saya ingin agar antar-halaman tidak lagi menumpuk memanjang di bawah, melainkan berganti tampilan secara dinamis tanpa proses *reload* browser."

### 6. Desain Profesional Khas Pemerintahan & Auto-Save
"Tolong perhalus dan percantik UI halaman Login beserta halaman Dashboard agar terlihat jauh lebih elegan dan berstandar profesional seperti aplikasi instansi Pemerintah/Dinas. Gunakan font modern (Plus Jakarta Sans) dan warna gradasi yang rapi. 

Lalu sempurnakan fiturnya:
- Ubah indikator 'Login Aktif' menjadi tombol menu *dropdown* tanpa batas kotak yang jelek.
- Hilangkan tombol 'Simpan' pada tabel status kendaraan. Buat agar sistem melakukan *auto-save* (*trigger* otomatis ke database) langsung ketika admin mengubah pilihan status pada *select box*."
